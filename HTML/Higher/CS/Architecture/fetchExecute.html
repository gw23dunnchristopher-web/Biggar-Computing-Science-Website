The Fetch Execute Cycle

What is the Fetch Execute Cycle?

The Fetch Execute Cycle is the process that a computer uses to execute a program. It is a cycle because it repeats itself until the program is finished. Instructions are retreived from memory, decoded and then carried out. This is repeated for every instruction in the program. The cycle consists of 4 stages:

  • The processor sets up the address bus with the required address.
  • The processor activates the read line on the control bus.
  • An instruction is fetched from the memory location using the database and stored in the instruction register.
  • The instruction in the instruction register is then interpreted by the decoder and carried out.

Parts of the Processor Used

The parts of the processor are used in the Fetch-Execute cycle. Some of them are registers.

PartOperation
Memory Address Register (MAR)Stores the address of the current instruction being fetched.
Memory Data Register (MDR)Stores the instruction fetched from memory.
Current Instruction Register (CIR)Stores the instruction to be executed.
Program Counter (PC)Stores the address of the next instruction to be fetched.
Instruction DecoderDecodes the instruction to be executed.
AccumulatorStores the result of the instruction.

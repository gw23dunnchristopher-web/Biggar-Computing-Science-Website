You need to be aware of the following Encryption methods at higher level:

  • Asymmetric Encryption - a secure communication method where a public key encrypts data and a corresponding private key decrypts it.
  • Digital Signatures - a cryptographic method that ensures the authenticity and integrity of a digital document or message.
  • Digital Certificates - an electronic document issued by a trusted authority to verify the identity of a user, organization, or website.

Asymmetric Encryption

Encryption is a method of securing data by converting it into a code that can only be read by someone with the correct key. Symmetric encryption is where the key (cipher) used to encode the data is the same as the key used to decode it. However, at the Higher level you also have to be aware of Asymmetric Encryption, this is where the two ciphers, the one used to encdoe the data and the one used to decode it, are different.

In the case of Asymmetric Encryption, the cipher used to encode the data is called the Public Key, and the cipher used to decode the data is called the Private Key. Both of these keys are mathematically linked, so that the data encoded with the Public Key can only be decoded with the Private Key.

This means that the Public Key can be shared with anyone, and the data can be sent securely, as only the person with the Private Key can decode it. These Keys are typically large prime numbers, which makes it computationally difficult to decode the data without the Private Key.

Digital Certificate

This certificate is issued to a website by a trusted third party called a Certificate Authority (CA) to verify the website's authenticity. CAs include companies like Verisign, Entrust, Geotrust and Digicert.

When a user visits a website, the browser checks the website's digital certificate to ensure that it is genuine. If the certificate is valid, the browser will display a padlock icon in the address bar, and the website will be considered secure. If the certificate is not valid, the browser will display a warning message and the website will not be considered secure.

The digital certificate contains:

  • a digital signature that was encrypted using a private key to confirm the identity of the server.
  • the server's public key that is used to...
    • decrypt the server's digital signature.
    • encrypt the data that is sent.
  • details of the Certificate Authority (e.g. Digicert, Verisign, etc)
  • a uniquely identifying serial number, start date and expiry date

Digital certificates prove that a website is legitimate and secure, and this increases customer trust. The customer can be sure that any communication between themselves and the website is genuine and has not been intercepted or tampered with. This is why you see the padlock icon in the address bar of most commercial websites.

Digital Signature

Much like signing a paper document with your signature, the purpose of a digital signature is to guarantee that a digital document is actually from the sender and is not forged. It also ensures that the document has not been tampered with or altered in any way during the sending.

There are three algorithms used in this process:

  • Key generation algorithm - creates a pair of keys, a public key and a private key.
  • Signing algorithm - encrypts the data using the private key to create the digital signature.
  • Signature verification algorithm - decrypts the data using the public key to verify the digital signature proving that the sender is genuine and that the message has not been changed.

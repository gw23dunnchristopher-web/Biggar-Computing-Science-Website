What is Floating Point Representation?

A number with a fractional part is stored using floating-point representation. This is known as a real number. For example, 4.2341 or 0.31 would be real numbers.

The reason we need floating point representation is because a computer can only represent numbers using bits, and bits can only really be converted into whole decimal numbers, so we need a way for the computer to represent fractions.

To convert a number to floating point representation involves moving the point so the number is a fractional value, and then counting how many decimal places you have moved the point.

Three parts are stored - the signed bit, the mantissa and the exponent.

Signed Bit

A single bit used to indicate the sign of the real number (0 represents a positive number and 1 represents a negative number).
Mantissa

These are the numbers that are stored after the point, once the point has been moved to the left of the first digit that is greater than 0.
Exponent

This is the number of decimal places that the point has been moved to the left. The exponent is stored in binary, and the number of bits used to store the exponent will determine the maximum number that can be stored.

If the point has to be moved to the right (for example in the case of a number such as 0.001) then the exponent will be negative.

Bit Allocation

A number of bits will be allocated to each of the three parts of the floating point number. In the Computing Science exam, they usually use 16 bits for the sign bit and the mantissa, and 8 bits for the exponent, as shown in the example below.

This may not always be the case though, so ensure that you look to see how many bits are being allocated to each one!

Changing the Bit Allocation

If a greater number of bits are assigned to the mantissa then it increases the precision of the number. For example, 0.19283 would require more bits for the mantissa than 0.2.

If a greater number of bits are assigned to the exponent then it increases the range of the number. The maximum and minimum number that can be represented both increase.

Worked Examples

In an exam, questions like this will be worth three marks. You will get one mark for correctly showing the sign bit, one for the mantissa and one for the exponent.

Example 1

The decimal number 6.125 converted to binary is 110.001.

Convert 110.001 to floating point representation. There are 16 bits for the mantissa (including the sign bit) and 8 bits for the exponent.

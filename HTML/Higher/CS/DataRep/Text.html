Extended ASCII vs Unicode

Extended ASCII

  • American Standard Code for Information Interchange.
  • Used as the standard character code by many computers for decades. As it is used by many computers it means that it is easy to transfer data between computers.
  • 8 bits are used to represent each character. This means that there are 256 (28 possible characters. This includes non-printable control characters such as TAB or ENTER, and printable characters such as letters, numbers and punctuation.
  • Standard ASCII also exists, which uses 7 bits to represent each character. This means that there are 128 (27) possible characters.

Unicode

  • Unicode is a standard that allows computers to consistently represent and handle text expressed in most of the world's writing systems. It is an International version of ASCII that includes non-English alphabets.
  • So that it has enough space to account for the additional characters, Unicode uses 16 bits to represent each character. This means that there are 65,536 (216) possible characters, many more than ASCII.
  • However, using 16 bits as opposed to 8 bits means that any text file using Unicode would take up twice as much space in storage as one using ASCII.

Summary

ASCII Unicode
Advantages
  • 8 bits per character, so smaller file sizes.
  • Used by many computers, so easy to transfer data between computers.
  • Can represent all characters from all languages.
Disadvantages
  • Can only represent 256 characters.
  • 16 bits per character, so larger file sizes.

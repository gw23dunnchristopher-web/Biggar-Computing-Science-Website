Forms

The HTML <form> element defines a form that is used to collect user input:

<form action="">

<!-- form elements -->

</form>

HTML 5 can be used to create and perform client-side validation on forms, without the need for JavaScript. Form elements implemented at Higher level are <input> (with its type attribute and the text, number, radio and submit values), <textarea>, <select> and <option>.

Where appropriate, form inputs will be validated using length, presence and range checks. Here is an example form:

Input Tag Types

Text▶

The example below implements two text input boxes, including length and presence checks:

First Name:<br>

<input type="text" name="firstName" maxlength="20" required><br><br>

Last Name:<br>

<input type="text" name="lastName" maxlength="20" required><br>

The above code includes the following:

type="text" Identifies the input type of the form element as text.
name="firstName" The name attribute is required when a form's data is submitted to the server for processing. Although submitting a form to a server is note required until Advanced Higher, it is good practice to include the name attribute for all form elements.
maxlength="20" Specifies the maximum number of characters allowed in the input field.
required Specifies that the input field must be filled out before submitting the form.

Number▶

Number input may be limited to a minimum value, maximum value or both:

Number of Tickets Required:<br>

<input type="number" name="age" min="1" max="3"><br><br>

The above code includes the following:

type="number" Identifies the input type of the form element as number.
min="1" Specifies the minimum value allowed for the input field.
max="3" Specifies the maximum value allowed for the input field.

Radio Buttons▶

Radio buttons are used to select one option from a list of options:

Choose your age:<br>

<input type="radio" name="age" value="12 to 14">12-14<br>

<input type="radio" name="age" value="15 to 16">15-16<br>

<input type="radio" name="age" value="Over 16">Over 16<br>

<br><br>

When submitted, the above form would return the name attribute “age” along with one of the listed values: 12 to 14, 15 or 16, 17 or over. Although Higher forms are not submitted to a server, it is good practice to include both the name and value attributes.

To align the radio buttons vertically, place a <br> tag in between each <input> tag.

Submit▶

When a form is submitted, the browser shows any validation errors (check browser versions, as this is browser dependant).

To allow candidates visual acknowledgement that an action is performed when the submit button is clicked, a JavaScript onclick event can be applied to the button as shown below. Additionally, an id has been applied to the submit button here so that it can be centred using 'display:block; margin:auto;' in CSS:

<input type="submit" id="submitButton" value="Submit" onclick="alert('Form Submitted')">

Additional Tags Used in Forms

Textarea▶

A larger text box, for use with extended text input, can be implemented using the <textarea> form element:

If required, please state any special requirements:<br>

<textarea name="requirements" rows="5" cols="30" maxlength="100"></textarea><br><br>

<br><br>

The width and height of a <textarea> element is set using rows and columns. If required, a length and presence check can be applied to above input element.

Select and Option - Dropdown Menu▶

The <select> is used to create a list of possible inputs in the form of a drop-down menu. Input choices are placed inside <option> elements:

Select play:<br>

<select name="play">

<option value="Hamlet">Hamlet</option>

<option value="Godo">Waiting for Godo</option>

<option value="Blood Brothers">Blood Brothers</option>

<option value="Romeo and Juliet">Romeo and Juliet</option>

</select><br><br>

<br><br>

As previously stated, the name and value attributes are not required at Higher, but it is good practice to include both.

Dropdown Menu (with size attribute)

The size attribute can be used within the select element, to display a set number of options. If the number of options is larger than the size attribute, a scroll bar will automatically appear:

<select name = "play" size="3">

Dropdown menu (with the 'multiple' attribute)

The multiple attribute can be used within the select element, to allow multiple options to be selected by holding down the Ctrl button:

<select name = "play" size="3" multiple>

Pre-populating Form Inputs

To aid user input, form elements can be given values that are displayed when the web page loads. These can be left unchanged, deleted or edited by the user, when they are completing the form.

Example 1 - Text▶

The following code will display the text 'forename' in the first name input box:

<input type="text" name="firstName" value="forename" maxlength="20" required><br><br>

Example 2 - Number▶

The following code will display the number '1' in the number input box:

<input type="number" name="tickets" value="1" min="1" max="3"><br><br>

Example 3 - Textarea▶

To pre-populate a textarea element, the text is included between the start and end elements:

<textarea name="requirements" rows="5" cols="30" maxlength="100">None.< /textarea><br><br>

Example 4 - Radio Buttons▶

The checked attribute is used to select one of the radio buttons in a form automatically.

<input type="radio" name="age" value="12 to 14" checked >12-14<br>

<input type="radio" name="age" value="15 to 16">15-16<br>

<input type="radio" name="age" value="Over 16">Over 16<br>

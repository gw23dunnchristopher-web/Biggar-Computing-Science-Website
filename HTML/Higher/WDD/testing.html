Usability Testing

This kind of testing involves systematic observation of people using a product to determine who easily people can figure out how to use it. In the case of website design, the 'product' is a website, and usability testing will help the website developer see how easy the website is to use. The goal is to recreate real world scenarios - the tester will be given a task that they have to complete on the website without implicitly telling them what to do. Then, by observing their behavior, the developer will be able to figure out if the website needs to be changed or modified so that it becomes easier to use for newcomers. Normally, low-fidelity prototypes are used in usability testing, with testers using a paper based version of the website. This means that any issues with the design can be hopefully be resolved before the website is coded.

Testers may be given:

  • A persona - a fictional user profile, usually backed by data from whichever company owns the website. Personas allow testers to represent various different demographics including, age, ethnicity, technical experience, even environmental constraints (such as slow internet).
  • Test Cases - A set of actions that the tester must be executed during the test to verify a particular feature or function of the website. For example, 'Tester must be able to log in to the website using their email address and password'.
  • Scenarios - A fictional set of goals that the tester must achieve. These are usually written in the form of a scenario, such as 'You are looking for a new job, and you want to find a job that is located in Glasgow'. The tester will then have to use the website to find a job in Glasgow.

The testers use the low-fidelity prototypes under a variety of conditions, during which they are observed. The observers make notes about any difficulties that the testers experienced and what alterations are required to the website design to make it easier to use the website.

Post-Implementation Website Testing

There are a number of tests you should carry out on your website to ensure that it meets the functional requirements.

Input Validation

This involves checking that any forms on your website work correctly. For example:

  • Your form should not accept text typed into a number field.
  • If one of the fields in your form requires numbers within a certain range, it should not accept numbers outside that range.
  • Fields that should only accept a certain number of characters should not accept any data that contains more than the maximum amount or less than the minimum amount.
  • Any fields that should be pre-populated with data have data in them when the webpage loads.

Navigation Testing

The navigation on the website should be tested to ensure that:

  • the links on the navigation bar take you to the correct pages.
  • all of the external links work correctly.
  • you can return to the home page from any of the other pages on the website.
  • all of the internal links work correctly.
  • there are no 'orphan' pages (pages that are not connected to any other page).

Media

This involves testing all of the images, text, and videos on your website to ensure that they all work correctly and appear in the correct positions.

Compatability Testing

Compatability testing involves testing the website on different browsers and devices to ensure that it works correctly on all of them.

Different browsers may use different versions of HTML and CSS, so it is important to test the website on all of the browsers that you expect your users to use.

Different devices may have different screen sizes, and this may cause the website to display differently. It is possible to emulate different screen sizes on a computer so that you can see what the website would look like.

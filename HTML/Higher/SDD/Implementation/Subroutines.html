Functions and Procedures

At Higher level, programs that you create must be modular, meaning that the must be broken down into different subroutines. The terms sub-program, subroutine, procedure and function are used by computer scientists to describe an independent block of code within a program. Each block can be reused, if necessary, either within the same program or as part of a module within a module library. When a main program ‘calls’ a sub-program it will run the code contained within the sub-program.

To be safe, in an exam, it is always wise to refer to functions and procedures as sub-programs or subroutines unless asked about them specifically.

  • A sub-program, or subroutine, is a block of code that can be called and accessed by a main program.
  • Procedures are subroutines designed to perform a series of commands with values sometimes passed from or to another part of the program.
  • Functions are subroutines designed to always return a single value to another part of the program.
  • In Python only a function may return more than one value.

Example of a Function

This is an example of a function created to find the maximum value in a list:

def findMax(testScore):

max = testScore[0]

for x in list:

if x > max:

max = x

return max

When called, this function will return the value stored in the local variable called 'max'.

Calling a Function

When a function is called, the value that it returns must be stored in a variable. For example, the function on the example above could be calleed as follows:

maxScore = findMax(testScore)

In this example, the value returned by the function is stored in the variable called 'maxScore'.

Note that, in Python, it is also possible for a Function to return multiple values, these values would each have to be stored in a separate variable. Below is an example of a function that would return multiple values, as well as how it would be called.

def findMinMax(testScore):

max = testScore[0]

min = testScore[0]

for x in list:

if x > max:

max = x

if x < min:

min = x

return min, max


minScore, maxScore = findMinMax(testScore)

The purpose of this function is to return the maximum and minimum scores in the testScore array. You can see that when it is called it stores the maximum and minimum values in the variables minScore and maxScore. The variables are separated by a comma and typed in the same order as the values are returned.

Example of a Procedure

This is an example of a procedure created to display the values in a list:

def displayList(testScore):

for x in list:

print(x)

When called, this procedure will display the values stored in the array called testScore.

Calling a Procedure

When a procedure is called, it does not return a value, so it does not need to be stored in a variable. For example, the procedure on the example above could be calleed as follows:

displayList(testScore)

Parameters

A parameter is a special kind of variable that used in a function or procedure to refer to one of the pieces of data provided as input. These inputs are known as arguments when the function is called.

It is important to remember that local variables are essentially placeholders for the arguments that are passed to the function. This means that the local variable will take on the value of the argument that is passed to it. If the argument passed in is a variable, then it could have a different name from the corrosponding parameter. This means that:

  • subroutines can be re-used with different arguments.
  • subroutines from module libraries do not need to be edited to account for different arguments.
  • different people can write subroutines that work together.

Actual Parameters

This is the value that is passed to the subroutine when it is called. The terms 'actual parameter' and argument' are often used interchangably, but there is a subtle difference between the two. The actual parameter can refer to the name of a variable that is passed into a function, but the argument refers to the value that is being passed into that function.

Formal Parameters

This is the parameter that is stated when the subroutine is declared. In many programming languages we can control how data flows into the subroutine by passing this parameter by reference or by value.

If it is passed by value, that means that what the subroutine actually handles is a 'copy' of the value passed into it. This means that the original value will not be affected if the copy changes within the subroutine.

However, if it is passed by reference, then the subroutine will reference the original value passed into it, and any changes made to that value will change the value in the main program.

Python always uses a third method called 'pass by assignment' which is similar to pass by reference, but not the same. With this method, if a value such as a string or integer is passed into the subroutine then it will receive a copy of the data and the original data will not be affected. However, if a list or array is passed in, then any changes made to the list or array within the subroutine will affect the original.

Example of Formal and Actual Parameters

In the example below, the formal parameters are shown in green and the actual parameters shown in red.

def addNumbers(num1, num2):

return num1 + num2


first = 14

result = addNumbers(first, 10)

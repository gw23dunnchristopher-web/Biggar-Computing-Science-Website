Variable Scope

The 'scope' of a variable determines the parts of the program where the variable can be accessed. You can limit the scope of a variable by:

  • creating a local variable that is declared inside a subroutine.
  • creating a formal parameter which is useable inside a subroutine

Global Variables

  • Declared outside of subprograms.
  • Accessible by every line of the program - main program and every sub-program.
  • In RAM throughout execution of the entire program - causing a greater demand on system resources.
  • Dangerous for programmers as can be “seen” by the whole program and its name, e.g. total, may “overlap” with a variable elsewhere and cause confusion.

Local Variables

  • Declared inside a subprogram.
  • Only accessible within the subprogram that it has been declared in.
  • Only present in RAM during the execution of the sub-program where they are located - more efficient in terms of demands on system resources.

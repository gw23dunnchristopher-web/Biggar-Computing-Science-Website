SDD Analysis

At the N5 level, in the Analysis phase we had to identify the purpose of the software as well as the functional requirements (the inputs, processes and outputs). At Higher level, we expand on that and, in addition to the functional requirements, we also have to identify the Boundaries and the Scope of the software.

Elements of Analysis

Purpose

  • A general description of the purpose of the software.
  • Called the software specification - basis of a legal contract between the client (customer) and software company.

Scope

  • A list of what will be handed over to the client or end-user, for example:
    • Design documents
    • Completed program
    • Test plan and test results
    • Evaluation report
    • Information on how long the project will take to create

Boundaries

  • Defines the limits of the software, what is in the project and what is not.
  • Clarification of any assumptions made by the software developers about the client's requirements.

Functional Requirements

  • Inputs, processes and outputs.
  • Any other features of the software or functions that must be delivered by the system.

Example of Software Analysis

Purpose The purpose of this program is to take 20 pupil names, their prelim marks and their assignment marks from a file. Calculate the percentage, and then find and display the name and percentage of the pupil with the highest percentage.
Scope This development involves creating a modular program. The deliverables include:
  • detailed design of the program structure
  • test plan with completed test data table
  • working program
  • results of testing
  • evaluation report
  • This development work must be completed within 4 hours.
Boundaries The program will read the pupil data (name, prelim mark and assignment mark) for 20 pupils from a sequential file. The data is accurate, so there is no need to implement input validation.

The pupil with the top mark will be the pupil who has the highest percentage. The only output needed is the name and percentage of the pupil with the highest percentage.
Functional Requirements These are defined in terms of the inputs, processes and outputs detailed below. All inputs are imported from a sequential file and all outputs displayed on the screen.

The program is activated by double clicking on the file icon and then selecting “Run” from the menu. Each process should be a separate procedure or function that is called from the main program.
Inputs Pupil name
Prelim mark
Assignment mark
Processes Calculate the percentage for each pupil
Find the name and percentage of the pupil with the highest percentage
Outputs Name of the pupil with the highest percentage

A player wins a hole if they have fewer shots than their opponent. For example, Claire has won the first hole taking only two shots compared to Tina’s four shots.

After nine holes, Claire has won four holes and Tina has won two.

Both players’ names and their nine scores are entered.

The app displays the name of the player who wins the most holes or a message stating the game has been drawn if the number of holes won is the same.

One boundary of this app is that the app is for games between exactly two players.

State two other boundaries for this app.

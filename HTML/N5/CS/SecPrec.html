Security Precautions

At the N5 level, when looking at security precautions for computers we need to be aware of two things - firewalls and encryption.

Firewalls

Before we talk about firewalls, we must be aware of how data is sent across a network. When large amounts of data need to be sent (such as a file or a stream), the data is broken up into smaller, manageable pieces called packets. These packets contain not only the actual data but also 'metadata', such as the destination address, the source address, and information to help ensure the data is delivered correctly.

The firewall is a device that is placed in front of the network and is responsible for inspecting these packets. It inspects both incoming and outgoing packets.

This allows the firewall to:

  • Restrict access to the network, preventing unauthorised access.
  • Consider the source of the data packets and what type of data they contain.
  • Discard unauthorised packets, effectively filtering unwanted or dangerous data.
  • Allow traffic from common programs such as email or web browsers.
  • Prevent access to certain websites or services to prevent employees from accessing non-work resources.

Encryption

We send sensitive information out over the internet all the time. Whenever we purchase a product on the internet, we have to send out bank details to the company we are buying it from. Whenever we send a private message, that message has to be sent across the network to the person receiving it. Naturally, we don't want hackers to intercept this data and gain access to it, so the data is changed into a code, making it unreadable to anyone but the intended recipient. This process is known as encryption.

  1. A public key (a large prime number) which is freely available is used to encode the data.
  2. The encoded data is then transmitted across the network.
  3. A private decryption key (which is never shared) is used by the recipient's computer to decode the data.

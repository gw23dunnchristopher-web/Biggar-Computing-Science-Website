Data Representation - Text

ASCII

Computers do not understand letters like we do, but they do understand numbers. This is why, in order for a computer to understand letters, they have to receive a number and the check a table to see which letter that number would represent.

In the early days, there were a lot of different tables that computers could use for this, but this was very confusing. The letter P could be the number 27, or 48! So a standard list of numbers and letters was produced in America that all computers would use from that point on.

This list was known as ASCII or the American Standard Code for Information Interchange.

Although there are now a number of different lists that computers can use for different maximum numbers of characters, the first 256 characters are always the same as the characters on the ASCII list.

The common list allows all computers to understand text files from other computers.

Standard vs Extended

There are two kinds of ASCII - Standard ASCII and Extended ASCII.

Standard ASCII - Standard ASCII uses 7 bits to represent each letter. The maximum number of letters that can be represented using standard ASCII is 128 (1111111 in binary is 127, but since 0000000 can also represent a character, there are 128 possible characters).

Extended ASCII - Extended ASCII uses 8 bits to represent characters, enabling it to represent 256 characters (11111111 or 255 characters plus 1 for 00000000).

Printable Characters and Control Characters

ASCII needs to be able to represent Printable Characters and Control (Non-Printable) Characters.

Printable Characters - These are characters that can be display on the screen, such as letters, numbers and symbols. A space is also a printable character, since it creates a clear block on the screen.

Control (non-printable) Characters - These cannot be displayed on the screen, and tend to perform special functions. For example: Enter, Backspace, Tab, Capslock, and Ctrl are all Control Characters.

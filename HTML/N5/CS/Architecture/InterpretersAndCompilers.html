Computer Architecture - Interpreters and Compilers

Programs are sets of instructions for the computer.

They are written using High Level Languages that contain English so that the programmer can understand them. Examples of High Level Languages include Python, Visual Basic, C++ and C#.

These High level languages must be translated into machine code (binary) in order for the processor to understand and execute them.

There are two types of translators used for this purpose - Interpreters and Compilers.

Compilers

  • A compiler translates the high level language to machine code in a single operation.
  • The original high-level program is called the source code.
  • Once the object code has been created, the source code is no longer required. It creates an executable file for the program that runs without the original coding environment.
  • It runs fast, but once if any changes are to be made it must be translated again.

Interpreters

  • An interpreter changes the high-level language into machine code line by line as the program runs.
  • Interpreted programs run more slowly than compiled programs, because the interpreter must translate each line every time the program is run.
  • However, the interpreter can also report any errors in the code right away as it runs.

Compilers vs Interpreters

Compiler Interpreter
Run fast. Run slow.
Report mistakes at the end of compilation. Report mistakes immediately.
Whole program is translated (compiled) at once. Code is translated one line at a time.
Translate and run are seperate processes. Translate and run is a single process.
Can save translated code as an executable. Cannot save the translated version of the code.
Compiler not required to run the code once it has been compiled. The Interpreter is required to run the code.

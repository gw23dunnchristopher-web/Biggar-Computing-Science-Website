How do we test a website?

Testing a website is an important part of the development process. It is important to test the website to ensure that it works as expected, and that it is accessible to all users. There are a numbe of different types of testing that can be carried out, including:

  • UI Testing - Making sure that the user interface matches the original design.
  • Hyperlink Testing - Making sure all of the pages are linked together correctly and are accessable, and the navigation works.
  • Media Testing - Making sure all of the media, inluding any images, videos and audio work as expected.
  • Consistency Testing - Making sure that all of the webpages follow a consistent theme and style, and look like they are part of the same website.

UI Testing

In the design stage, you will have created a wireframe that shows the layout for the website. When you test the user interface, you will compare your website to this wireframe and make sure that they match. While testing this, you should also look at the end-user requirements collected during the Analysis phase, and make sure that any accessibility requirements (accounting for the needs of users with colour blindness, etc. are met if nnecessary.

Hyperlink Testing

Your website may have a lot of different pages, and you want to make sure that all of them are accessible. You will have to test that all of the links work correctly on each page. To do this, the easiest method is to keep a note of all of the links on each page and check them off as you test them. If any of the links are broken, you can go into the code and figure out why they aren't working, fixing them as you go.

Media Testing

Just as with Hyperlink testing it is wise to make a checklist of images that you have used on your website and what pages you have used them on as you go. You can use this list to test each page on the website to make sure all of the images display correctly and all of the videos play as expected. The positioning of the images should match the design, if the images cannot be loaded, alternative text should appear. Video and audio should be able to be played, paused and restarted. Any autoplay video or audio play automatically as expected.

Consistency Testing

As you test your website, you should be checking that all of the pages follow the same theme and style. This includes the layout, the colors, the fonts, and the overall look and feel of the website. You should also check that all of the pages are consistent in terms of the navigation, and that the navigation is easy to use.

Past Paper Question - 2018 Q8

8

Describe one aspect of consistency that should be considered when testing a website.

 1

Any one from:

  • text size or font is consistent (across similar elements)
  • graphics are the same size
  • position of content/navigation bar is consistent across different pages/objects aligned
  • colour scheme consistent throughout.

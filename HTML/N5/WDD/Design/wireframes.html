Wireframes

What is a Wireframe?

User interface planning can be illustrated using wireframes - sketches of the user interface that show the layout of the page. They are used to ensure consistency between web pages and share details with members of the development team.

  • A separate wireframe is needed for each page on the website.
  • Each wireframe indicates the intended layout of the page and shows the position of:
    • all text elements on the page.
    • any media elements (images, audio clips and video clips).
    • elements that allow the user to interact with the page.
    • intended position and type of all hyperlinks on the page.

Webpage Elements on a Wireframe

A wireframe is just a sketch of a webpage, but there are commonly used symbols that are used to represent different elements.

Element Symbol Explanation
Text Information on this part of the website. Wireframes contain placeholders representing larger pieces of information. A text placeholder would tell the reader what the text in that part of the website would be about.
Headings

Heading

 Headings on a wireframe are the rough size and shape that they would be on the actual webpage.
Bullet Points
  • Bullet point 1
  • Bullet point 2
  • Bullet point 3
 Bullet points should show an example of what they are being used for, and show the position that they would be in.
Links Link When using text as a link for the user to click on, this text should be represented by underlining it.
Images Images are shown as a rectangle with a cross in it, this just shows the position of the image. If a particular image is being used, then the filename of the image will be shown.
Video Videos are usually represented by a rectangle with the play symbol, a triangle, in the middle. If there is a particular video that will be used, its filename is noted as well.
Audio Audio on the website is usually represented by a horizontal bar with a small speaker symbol. If a file is being used for this, the filename will be noted.

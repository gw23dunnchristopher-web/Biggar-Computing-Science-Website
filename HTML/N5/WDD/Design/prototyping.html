Prototyping

What is a Prototype in Web Development?

In terms of Web Design and Development, a prototype is an early version or model of a website designed to demonstrate its layout, functionality and user-interaction before full scale development begins. It acts as a blueprint to explore and refine ideas, identify potential issues and ensure the design aligns with user expectations.

Prototypes can range from low-fidelity to high-fidelity:

  • Low-fidelity prototype - These are simple sketches or wireframes, focusing on the basic structure and functionality. They are quick to create and useful for brainstorming ideas.
  • High-fidelity prototype - These are more detailed and interactive, resembling the final product with clickable links, visual elements, and sometimes even sample data. Tools like Adobe XD, Figma, or Sketch are often used to create these.

Low-fidelity Prototype

As hand-drawn sketches of the webpages, low-fidelity protoypes are the cheapest and easiest kind of prototype to make. Coloured pencils, felt pens and markers can be used to add colour, and quick hand drawings of images and widget icons will give end users a good idea of what is intended.

Any size of paper can be used but flip-chart paper can be easier for candidates to handle and facilitates collaborative group work.

There are many free prototyping tools that can be used if you don't want to draw the prototype by hand, some of them are online. Here are a few examples:

  • Figma
  • Proto.io
  • Sketch
  • Pixso

Templates can be used to create realistic looking interfaces that can be exported as PNG files and printed to generate prototpyes for usability testing.

Graphics packages such as 'Paint' can be used to create the intended layout using tools provided in the toolbox.

Usability Testing

This is a kind of testing performed on low-fidelity prototypes before work on coding the website actually begins.

  • End-users or testers complete some tasks using the paper prototype. For example: They could be asked to try logging in, finding something on the website.
  • Developers will physically move the paper prototype during the test to change the screen or update its content.
  • Feedback from testers and observations help developers to make changes and improvements to the user interface at an early stage in development.
  • More of the details of the content and the screen widgets are shown on the prototype than in a wireframe.

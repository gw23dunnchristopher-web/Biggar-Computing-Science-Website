Evaluation

How do you evaluate a database?

Once you have implemented your database, it must be evaluated to make sure that it works as expected and meets the requirements that was set out in the Analysis phase.

If all of the requirements are met, then the database will be considered 'Fit for Purpose'. You will be able to not that some of the requirements have been met in the Testing phase.

Output Accuracy

During the testing phase, you should have been able to test that any of the SQL statements that you entered produced accurate resultes.

If there is a difference from what you expected, then you have to consider why there is a difference and how it can be fixed.

  • Logic Error - Your SQL queries were intered incorrectly, or used incorrect criteria, causing an incorrect output.
  • Data Error - The data in the database was incorrect, or there was a mistake in the data entry.

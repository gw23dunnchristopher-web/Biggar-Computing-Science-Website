Databases Introduction

What is a database?

A database is an organized collection of data that can be easily accessed, managed, and updated. Think of it as a digital filing system, where information is stored in a structured way so it can be quickly retrieved or modified when needed. Databases are used in nearly every aspect of technology, from websites and apps to financial systems and beyond.

Before computers became common in the workplace, it was common to use paper-based databases. We used filing cabinets for this, and whenever we needed to get information from the database we had to open the filing cabinet and manually look through the files contained inside to find the information we needed.

Nowadays, we use computers to store and manage our databases, this has a few benefits:

  • It is much faster to search and sort the information in an electronic database.
  • Electronic databases help to ensure that there are no cases of duplicate data.
  • In paper databases, it was easier to incorrectly file a record, and only one person could look at a record at a time.
  • Electronic databases take up less space than computer databases.

Information and Data

While when might use the words information and data interchangeably in everyday language, they are very different things in the context of databases.

  • Data is the raw facts and figures, numbers and text that have no meaning on their own.
  • Information is the data that has been processed and given meaning.

For example, the data '200' has no meaning on its own, but the information '200 students' does.

The Structure of a Database

Databases are set out in Files, Records and Fields.

Structure Description Example
File A file is a collection of structured information on a specific topic. Information on all of the students in the school.
Record A record is a collection of structured information on a particular person or thing within that topic. A file contains many records. Information on a single student.
Field A field is a single piece of data which is contained in each record. Records are made up of, usually, 2 or more fields. A record contains many fields. The student's first name.

In this example, the whole table is a File, the rows are Records and the columns are Fields.

Flat File vs Relational Databases

A Flat File database contains only one table (file), and a Relational Database contains two or more tables that are linked together in a relationship. When considering large sets of data, it is better to use a Relational Database rather than a Flat File Database because:

  • It breaks the data down into smaller, more manageable data sets.
  • It is easier to update a Relational Database. For example, if you have a database of people and cars, and you need to update a person's address you would just have to update it once on the person table rather than having to do it for every car.
  • It avoids duplication of data, such as in the car example where you would have to enter the name of the owner over an over in a Flat File Database.

Primary Keys

Each record must have a field dedicated to uniquely identify that record - this is known as the Primary Key. The data stored in this field will be unique to each record. For example, a database of customers could have an customer ID for each person, this would be used as the Primary Key for those records.

Foreign Keys

When you have a relational database with two or more tables, Foreign Keys will be used to link the tables together. A Foreign Key is the Primary Key of one table which has been copied into another. For example, if a customer owns many cars then thier Customer ID might be used as the Foreign Key in a Cars table. Their Customer ID on the record of any car that they own in the table, linking the information about that customer to each car.

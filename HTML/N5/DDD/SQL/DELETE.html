SQL DELETE Statements

DELETE statements are used to delete data from a database. A DELETE statement consists of 2 main parts:

  1. Specify the table that has the information to be deleted.
  2. The criteria for which records to delete.

Deleting a single record

Imagine that the ride "Nemesis" has been deemed unsafe and has been closed down. It will need to be removed from the database.

Step Action Code
1 Specify the table that you want to delete the record from.

DELETE FROM Ride
2 Specify the criteria for which record to delete. In this case, we want to delete the record where the ride name is "Nemesis". There is only one ride called "Nemesis" on the list, so we do not have to worry about deleting other records by doing this.

DELETE FROM Ride

WHERE RideName = 'Nemesis';

Be careful when using the DELETE clause, as it is easy to accidentally delete additional records if you use the wrong criteria.

Deleting multiple records

Imagine that Park 1 has been closed for renovations and will have all new rides when it reopens. All of the current rides in Park 1 will have to be deleted from the Ride table.

Step Action Code
1 Specify the table that you want to delete the record from.

DELETE FROM Ride
2 Specify the criteria for which records to delete. In this case, we want to delete the records where the parkID is 'Pk1'.

DELETE FROM Ride

WHERE parkID = 'Pk1';

This will delete 6 records from the database, all of the records where the parkID is 'Pk1'.

Claire is no longer able to run the 3-4 Years class on a Tuesday in the Autumn block.

The SQL statement below is written to make the change.

DELETE FROM Class

WHERE leaderName = "Claire"

AND className = "3-4 Years"

Give one reason why this SQL statement is not fit for purpose.

 1

The SQL statement does not specify the day of the week or the block that the class is taking place in.

This means that it will delete all records where the leaderName is Claire and the className is 3-4 Years, regardless of the day of the week or the block.

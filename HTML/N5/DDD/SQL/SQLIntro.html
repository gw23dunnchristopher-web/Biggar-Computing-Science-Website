SQL Introduction

What is SQL?

SQL stands for Structured Query Language. It is a programming language that is used to manage and manipulate relational databases. It enables the user to perform a variety of tasks in a database such as querying data, creating and modifying database structures. SQL is widely used across various database management systems. Even MS Access is just a graphical user interface covering an SQL based system. Whenever you create a table, perform a search or sort data in Access, it actually runs the SQL code to do so in the background.

It is pronounced either S-Q-L or "sequel".

Microsoft Access provides a graphical user interface to query our database without using SQL using the "Query Design View" and the "Query Wizard".

Any query created with these tools can be viewed in the SQL view to examine the code that Access generated. At National 5 level you will be expected to create queries without using the Graphical User Interface.

How can you access SQL code?

You can access SQL code either from within MS Access or using an online SQL editor. Both of these methods will be covered below.

Using SQL in MS Access

If you want to access SQL from within Access, you first have to create a query, by clicking on the Create Tab at the top of the screen and then clicking on Query Design.

An SQL View button will appear in the top left corner of the window, which you can click on to enter the SQL View.

The SQL View is where you can type in SQL code to perform tasks in your database. If you open it after having just generated a blank query, it should look like this:

After typing the code you want to run in the SQL window, you can click on the Run button in the top left of the window to run it.

Using SandboxSQL

SandboxSQL is an online SQL editor and compiler. It uses SQLite as its language, and it is completely seperate from MS Access. This means that it does not use the Access '.accd' files, and instead uses '.sqlite' files. However, if you want to exclusively practice SQL code, this is a good way to do it without using access. SQL Code

When creating queries in SQL, you should take note of the following:

  • SQL Code could all be written on a single line, in the tutorials it is split up onto multiple lines to make it easier to read.
  • A full SQL query always ends in a semi-colon (;). Look at the example below and note that there is only one semi-colon at the end.
  • SQL code is not case sensitive. This means that you can type in the code in any case that you want, and it will still work.
  • However, to make it easier to read, it is good practice to capitalise keywords like SELECT, WHERE, and FROM.

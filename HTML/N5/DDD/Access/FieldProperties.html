Field Types and Properties

In the Design View of MS Access, whenever you have a field selected the properties of that field will appear at the bottom of the page. Different types of field have different properties, so if you want an explanation of how changing these properties will affect the field, click on one of the field types below.

  • Short Text


    Short Text fields are the most common type of field, they are used for text that is less than 255 characters long. The properties of a Short Text field that you will need to use at the N5 level are:

    • Field Size - This is the maximum number of characters that can be entered into the field. The default is 255, but you can change this to whatever you need.
    • Default Value - This is the initial value that will be entered before the user types anything, usually this is blank.
    • Validation Rule - This is the rule that will be used to validate the text entered into the field.
    • Validation Text - This is the text that will be displayed if the validation rule is not met.
    • Required - This is whether or not the field is required. The default is No, but you can change this to Yes if you need the user to enter something in this field to create a record.

  • Number


    Number fields are used to store numerical data, the type of numerical data can be adjusted in the properties. At N5, you need to be aware of:

    • Field Size - In a number field this is set to 'Long Integer', which is a number with no decimal places ranging from -2,147,483,648 to +2,147,483,647. This can be set to:
      • Single - a decimal number that can store up to 7 decimal places (uses 4 bytes).
      • Double - a decimal number that can store up to 15 decimal places (uses 8 bytes).
      • Decimal - a decimal number that can store up to 28 decimal places (uses 12 bytes).
    • Default Value - This is the initial value that will be entered before the user types anything, usually this is 0 in a number field.
    • Decimal Places - This sets the number of decimal places that the number entered will be rounded to. Initially this is set to Auto, which means it will not round the number until the maximum number of decimal places for the field size is reached.
    • Validation Rule - This is the rule that will be used to validate the number entered into the field.
    • Validation Text - This is the text that will be displayed if the validation rule is not met.
    • Required - This is whether or not the field is required. The default is No, but you can change this to Yes if you need the user to enter something in this field to create a record.

  • Currency


    Currency fields in access are a type of number field - they can be used to store amounts of money, but they can also be used to store percentages. At N5, the properties you should be aware of are:

    • Format - Allows you to change which kind of currency is being used. By default it is set to £ (UK pounds), but it can be set to Euros, percentages, or scientific notation.
    • Decimal Places - This sets the number of decimal places that the number entered will be rounded to. Initially this is set to Auto, which is 2 for the currency field, but it can be set to go all the way up to 15.
    • Default Value - This is the initial value that will be entered before the user types anything, usually this is 0 in a currency field.
    • Validation Rule - This is the rule that will be used to validate the amount entered into the field.
    • Validation Text - This is the text that will be displayed if the validation rule is not met.
    • Required - This is whether or not the field is required. The default is No, but you can change this to Yes if you need the user to enter something in this field to create a record.

  • Date/Time


    Date/Time fields store dates or times. At N5 you will need to be aware of the following properties:

    • Format - Allows you to specify how the date or time being entered is displayed, you will have access to the following options:
      • General Date - 16/04/2025 15:24:00 (The date followed by the time in hours, minutes and seconds)
      • Long Date - 16 April 2025 (The date fully typed out)
      • Medium Date - 16-Apr-25 (The day with the month abbreviated)
      • Short Date - 16/04/2025 (Just the date, no time)
      • Long Time - 15:24:00 (The time fully typed out in hours, minutes and seconds)
      • Medium Time - 03:24 PM (The time with AM and PM)
      • Short Time - 15:24 (The time in hours and minutes)
      If this is not set, then it will allow you to enter a date and/or a time in any format.
    • Default Value - This is the initial value that will be entered before the user types anything, usually this is blank.
    • Validation Rule - This is the rule that will be used to validate the date or time entered into the field.
    • Validation Text - This is the text that will be displayed if the validation rule is not met.
    • Required - This is whether or not the field is required. The default is No, but you can change this to Yes if you need the user to enter something in this field to create a record.

  • Yes/No


    Yes/No is MS Access's version of a Boolean field, in the Access Datasheet it will be represented by a checkbox. If the box is checked then that means 'Yes', otherwise it means 'No'. You have to be aware if the following properties for this field type at the N5 level:

    • Format - This can be set to Yes/No, True/False or On/Off. By default this will be 'Yes/No'.
    • Default Value - You can set it so that the checkbox is checked or not checked when the record is created. By default this will be set to 'No'.

    While there are options to validate a Yes/No field, you will not use them at the N5 level.

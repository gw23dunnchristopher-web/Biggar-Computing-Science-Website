Data Validation

What is data validation?

Data validation is the process of ensuring that the data entered into a database is valid. This is done by setting rules that the data must follow, if the data does not follow the rules then it will be rejected and the user will be asked to enter the data again.

There are four types of data validation that you need to be aware of at the N5 level:

  • Presence Check - Checks to make sure the user has entered data in the field before it allows them to create a record.
  • Range Check - Checks to make sure that the number entered is within a certain range.
  • Length Check - Checks to make sure that the user has entered a certain number of characters.
  • Restricted Choice - Gives the user a list of options to choose from.

This page will give you a tutorial on how to set up each type of validation in MS Access. Click on the buttons below to get information on each one.

Data Validation Tutorials

  • Presence Check

    1. Open your database in the Design View.

    2. Click on the Short Text field that you want to make required.

    3. In the properties for that field, change the 'Required' value to 'Yes'.

    4. Return to the datasheet view, you will be asked to save your table. Click 'Yes'.

    Now, if you try to create a record without entering anything in that field, it will give you an error.

  • Length Check

    Setting the field length of a field in its properties will prevent the user from entering any more than that number of characters. However, you may want the user to enter a certain number of characters. In Access, we can do that using the 'Len()' function.

    1. Open your database in the Design View.

    2. Click on the Short Text field that you want to make set the minimum length for. In this example, I have selected the field for the Designer's phone number, which must be 11 characters long as phone numbers are 11 numbers with 0 at the start.

    3. In the properties for that field, click on the validation rule box and type in Len([Phone Number]) = 11. The Len() function checks the length of anything in between the brackets. In this case we are checking the Phone Number field - the field name is typed in between square brackets. We are saying to the program that the length of the data in the specified field must be exactly equal to 11.

      You do not have to use =, if you want the user to enter at least a number of characters you could use > instead.

    4. Below the Validation Rule, in Validation Text, you will enter an error message that will appear if the user attempts to enter something other than what you have specified. In this case, the message will say that the user must enter 11 characters for a phone number.

    5. Return to the datasheet view. You will be asked to save your table.

    6. You may see the following messages if you have already typed something into your table.

      This just means that existing data may not meet your new rule, you can click 'Yes' to both of these.

    Now, if you try to enter an incorrect number of characters in this field it will display the error message that you created in Validation text.

  • Range Check

    Much in the same way that we edited the validation rule to do a length check, we can also edit it to do a range check for a number field. The table below shows the different symbols that you can use in thie Validation Rule box to check if the number beinging entered is withing the range you want. There are also some examples of validation rules that you could use.

    Symbol/Phrase Description Example Usage
    > Greater than > 0 This could be used to ensure that a a number less than 0 isn't entered for an age.
    < Less than < 120 This could be used to ensure that a number greater than 120 isn't entered for an age.
    >= Greater than or equal to >= 0 This could be used to ensure that the number entered for a quantity is greater than or equal to 0.
    <= Less than or equal to <= 100 This could be used to ensure that the number entered for a quantity is less than or equal to 100.
    Between Between two numbers Between 0 And 100 This could be used to ensure that the number entered for a quantity is between 0 and 100.

    You can follow these steps to set up a range check. In this example, we will ensure that the amount of items entered into the Items table is greater than or equal to 0, because there can't be a negative stock of items.

    1. Open your database in the Design View.

    2. Click on the number field that you want to validate with a range check.

    3. In the properties for that field, enter the validation rule into the 'validation rule' box. In this case, we want all numbers entered to be greater than or equal to 0, so we will enter ">=0".

    4. Below the Validation Rule, in Validation Text, you will enter an error message that will appear if the user attempts to enter something other than what you have specified. In this case, the message will say that says that the amount entered must be greater than or equal to 0..

    5. Return to the datasheet view. You will be asked to save your table.

    6. You may see the following messages if you have already typed something into your table.

      This just means that existing data may not meet your new rule, you can click 'Yes' to both of these.

    Now, if you try to enter a number into the field with this validation that is outside of the range you are checking for, you will get an error.

  • Restricted Choice

    Creating restricted choice validation means creating a dropdown box that the user will have to select from when they try to add data to a field in your table. In the example below, we will add restricted choice validation to the Type field in the Items table, so that the user must select from 'Chairs, Desks or Tables'.

    1. Open your database in the Design View.

    2. Change the data type of the field that you want to impose restricted choice validation on to Lookup Wizard.

    3. In the window that appears, choose "I will type the values that I want" and click "Next".

    4. In Column 1 of the window that opens, type the options that you want the users to choose from. I will type in "Chairs", "Desks" and "Tables". Click "Next"

    5. On the next screen, click on 'Finish'.

    6. Return to the datasheet view. You will be asked to save your table.

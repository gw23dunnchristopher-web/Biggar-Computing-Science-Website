Pseudocode

What is pseudocode?

Pseudocode is a text based form of design notation that is used to help programmers understand the logic of their code. It uses plain English to describe the steps of a program. Typically, the steps are listed in a numbered order, with each step being a single line of code. Sometimes a step can be refined down into further steps. For example, step 5 could become 5.1, 5.1 and 5.3. In N5, you must be able to understand programs designed in pseudocode, and it is highly recommended that you also learn to write pseudocode for questions in your exam where you have to use design notation.

Why is pseudocode used?

  • It is easy to read.
  • It does not rely on syntax, so it can be converted to any programming language.
  • It helps with problem-solving, as breaking a problem down into steps mirrors computational thinking.

Pseudocode Example

The following is pseudocode for a program that calculates the area of a rectangle. If the area of the rectangle is greater than 100, then the program will output "Big rectangle". If the area of the rectangle is less than or equal to 20, then the program will output "Small rectangle". If it is in between then it will output "Average rectangle".

Algorithm
1.Get the rectangle's height and width from the user.
2.Calculate the rectangle's area.
3.Determine if the rectangle is large or small and display this information with the total area.
Refinements
1.1Display "Enter rectangle's height in cm:".
1.2Get rectHeight from keyboard.
1.3Display "Enter rectangle's width in cm:".
1.4Get rectWidth from keyboard.
2Set area to rectHeight x rectWidth
3.1If area > 100 then:
3.2Display area + "cm^2. That's a big rectangle!"
3.3Else if area <= 20 then:
3.4Display area + "cm^2. That's a small rectangle!"
3.5Else:
3.6Display area + "cm^2. That's an average rectangle!"
3.7End if

You can see here how the basic steps for the program are written out at the top. These steps are called the 'Top Level Design'. The steps are then broken down further into refinements.

The following two questions are from past papers, and can be solved by reading and writing in pseudocode.

Past Paper Question 1 - 2022 Q8(b)

The design below shows how the total cost of a booking is calculated.

Algorithm
1.Find ticket Costs
2.Get quantity of passengers
3.Calculate initial cost of booking
4.Update cost of booking if bag(s) are added
5.Display final cost of booking
Refinements
2.1Get quantity of adult passengers
2.2Get quantity of child passengers

The app provides an option to add bags to the booking. Each passenger is asked if they want to add a bag. The cost is an additional £7 for each passenger who decides to take a bag.

Using a design technique of your choice, refine step 4.

Show Answer
3.1Loop for the number of passengers
3.2Ask if passenger wants a bag
3.3If answer = "Yes" then:
3.4Add £7 to bag total
3.5End If
3.6End Loop
3.7total = total + bag total

The program will loop a number of times equal to the number of passengers there are. For each passenger, it will ask if they want a bag. If they do, it will add 7 onto the total cost of the bags. Then the if statement and the loop will end. Finally, the amount required for the bags will be added onto the total amount for the booking. There are actually a couple of other ways to complete this problem. You could add up the number of bags and multiply that number by 7 at the end, for example.

Past Paper Question 2 - 2023 10(b)

The design below shows how a customer’s average monthly data usage is calculated and displayed.

Algorithm
1.Store each month's data usage
2.Calculate the average monthly data usage
3.Display the average monthly data usage

The data structure month is used to store the data usage.

The variable aveData is used to store a customer’s average monthly data usage.

Using a design technique of your choice, refine step 2.

Show Answer
2.1Loop 12 times
2.2Set total to total month [index]
2.3End loop
2.4Set aveData to total/12

The program loops 12 times, because there are 12 months. For each month, the prgoram adds the total amount that month to that overall total. After that, the loop ends and the overall total is divided by 12 to get the monthly average amount.

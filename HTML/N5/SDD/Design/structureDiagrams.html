Structure Diagrams

A structure diagram is another type of graphical notation used to create the design for programs. The symbols used are a little different to the ones used in a flowchart.

At the N5 level, you will be expected to be able to read a structure diagram. This is how to read a structure diagram:

  1. Look for the Main Block - At the top you'll usually find a large box or circle. This is the main part of the program or problem that you need to solve.
  2. Identify the Program's Parts - Branching off from the main component, you'll see smaller blocks or boxes. Each box usually represents a specific task or feature of the program.
  3. Follow the Connections - Lines link the different parts together, showing how they interact. Look for these lines.
  4. Check the heirarchy - Each of the boxes branching off from the main box may have smaller boxes connected to them, these are a further breakdown of the task in the box.
  5. Work from top to bottom, left to right - Starting at the top, move down to the first box on the left then, if it has onnected boxes, work through those from left to right first. Continue until there are no boxes left.

Structure Diagram - Symbols

Symbol Name Meaning
Process Used for a function, input, output or a calculation.
Terminal Used when instructions are to be repeated.
Initialisation Different actions may be taken depending on conditions being true (IF or CASE in code)
Input/output A pre-defined function will be used e.g. random, length or round

The Iterative Development Process

The method of software development you need to know at the N5 level is the traditional one - the iterative development process (also known as the 'waterfall' method. If consists of six basic stages:

Analysis In this stage, the primary goal is to understand the problem or need that the software will address. The functional requirements of the program are identified here.
Design The design stage is about creating the blueprint for the program - pseudocode, flowcharts or structure diagrams are used to lay out how the program will operate.
Implementation This is the phase where the code is written. Developers follow the design documents and specifications from the earlier stages to build the software system.
Testing In this phase, the code is tested to make sure that it works as required.
Documentation In this phase, the technical specifications and user guide documents are created. At N5 we will generally skip this phase.
Evaluation The program is assessed to see whether it meets the functional requirements and the initial design. If it does then it is considered to be 'fit for purpose'.

This is an iterative process - what that means is that you may need to return to an earlier stage because you realise that there is a problem with the program.

For example, when testing the program during the testing stage, you may need to return to the implementation stage.

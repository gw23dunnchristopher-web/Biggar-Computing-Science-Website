Evaluation

What is Evaluation?

The Evaluation stage of Software Development is when you look at your program and assess how well it meets its intended purpose and requirements. In this stage we look at four aspects of your program:

  • Fitness for Purpose
  • Efficiency
  • Robustness
  • Readability

Fitness for Purpose

For your program to be considered 'Fit for Purpose', it must meet the purpose and functional requirements that were written during the Analysis stage. You can use the purpose and the inputs, processes and outputs of the program as a sort of checklist to see if it meets the requirements.

Efficiency

Your program may complete the task that it was designed to do, but it may not be as efficient as it could be. What this means is that the program may take longer to complete the task than it could with more efficient code.

Check to make sure that there are no unnecessary lines of code in your program - they occupy story space in memory and take up processor time.

Loops

Loops can be used to improve the effiency of your code, look at the example below:

total = 0
print("Enter a number:")
number = int(input())
total+=number
print("Enter a number:")
number = int(input())
total+=number
print("Enter a number:")
number = int(input())
total+=number
print("Enter a number:")
number = int(input())
total+=number
average = total / 4
print("The average is" + str(average)
total = 0
for x in range(4):
print("Enter a number:")
number = int(input())
total+=number
average = total / 4
print("The average is" + str(average)

These two sets of code perform exactly the same function, but the code on the right is more efficient as it uses fewer lines, is quicker to write and takes up less storage space in memory.

1-D Arrays

Below are two sets of code - both programs find the average of 4 numbers.

print("Enter a number:")
number1 = int(input())
print("Enter a number:")
number2 = int(input())
print("Enter a number:")
number3 = int(input())
print("Enter a number:")
number4 = int(input())
average = (number1+number2+number3+number4) / 4
print("The average is" + str(average)
number = [0] * 4
total = 0
for x in range(4):
print("Enter a number:")
number[x] = int(input())
total+=number[x]
average = total / 4
print("The average is" + str(average)

the code on the right uses an array to store the numbers entered rather than storing each one in an individual variable. This is more efficient as it, together with the loop, uses fewer lines of code.

Selection

Using nested IFs or CASE statements instead of lots of IF statements is efficient as it ensures that no unnecessary lines of code are executed.

Four IFs with complex condition Nested IFs with simple conditions
if mark > 70:
grade = "A"
if mark >= 60 and mark <= 69:
grade = "B"
if mark >= 50 and mark <= 59:
grade = "C"
if mark < 50:
grade = "D"
if mark >= 70:
grade = "A"
elif mark >= 60:
grade = "B"
elif mark >= 50:
grade = "C"
else:
grade = "D"
This program always carries out 4 comparisons, no matter what the mark is. This program carries out either 1, 2, or 3 comparisons, depending on the mark.

Logical Operators

Logical operators like AND, OR and NOT can be useful when creating complex conditions, instead of using multiple simple conditions.

  • The program below uses two simple conditions:
if X > 4:
if Y < 6:
quadrant = 2
  • This program uses one complex conditions, using fewer lines:
if X > 4 and Y < 6:
quadrant = 2

Sequence

What order you place code in can also effect your program's efficiency. For example, calculating a running total within the loop that is used when values are being entered by the user and assigned to an array instead of using one loop to fill the array and then another loop to calculate the running total.

Robustness

This is the term used for the ability of your program to handled exceptional (or invalid) data without crashing. If your program uses input validation to catch incorrectly entered inputs and then displays an error message to the user before asking them to type in their answer again, it could be considered to be more 'robust'.

Readability

Your code might be understandable to you, but what about to other people? Your code should be understandable when read by any programmer, and there are several features that can help you to ensure this:

Internal Commentary
  • Comments at the start to describe the program, identidy the author and state when it was written.
  • Comments should be used to describe tricky code.
Meaningful Variable Names
  • Variable names should describe what is being stored in the variable to make the reader aware of their purpose.
Indentation
  • Indent code inside an IF, CASE, FOR, REPEAT and WHILE loop to make it easy to read the code and see which lines are in the if statement or loop.
  • In Python, proper indentation is required or the program will not work correctly.
White Space
  • Gaps between sections of code make it easier to read.
